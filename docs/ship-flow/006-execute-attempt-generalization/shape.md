# Execute attempt generalization — Shape

## Delegated Autonomous Reshape

This final child delivers Epic 006's approved agent-native dogfood result only
after the plan vertical and recovery behavior have landed. It replaces the
old integration/full-regression ending with a bounded consumer adoption:
execute reuses the proven generic seam, exhausted work dispatches nobody, and
#21 is one UAT rather than a compatibility program.

The prior lane showed two scope traps: a broad full-suite failure could pull
unrelated implementation into the child, and #21 preservation could expand
into fixtures or future-RED bookkeeping. This slice keeps changed surfaces
first and makes both exclusions executable.

## Outcome and Scope

One real execute caller adopts the shared attempt seam without duplicating its
plan lifecycle. At exhaustion, routing completes before lease, attempt,
envelope, or worker creation. After landing, #21 runs once against its
unchanged technical plan and evidence as the end-value UAT.

Out: dispatcher or shape-confirm repair, generic scheduling, unrelated test
repair, #21 product redesign, XFAIL registry, future-RED registry, and a
repository-wide regression program owned by this child.

## Acceptance Criteria

1. A focused execute integration test exercises the same generic attempt seam
   proven by plan, with no execute-only lifecycle duplicate.
2. Threshold + 1 returns the routeable exhausted outcome while worker, lease,
   attempt, and envelope creation counters remain zero.
3. Exactly one #21 post-landing UAT consumes the seam with its technical plan
   and evidence unchanged; no compatibility or expected-failure registry is
   created.

## Timebox and Return Contract

Child cap: **2h**. Every dispatch is estimated at 60m, makes an explicit
finish-versus-return decision at 90m, and hard-stops at 120m with a durable
HEAD, owned paths, checks, and next command. Reaching 2h routes back to
shape/plan; verification is not compressed to fit.

## Hand-off to Design

Import the landed plan/recovery contract as immutable input and design only
execute adoption, pre-dispatch exhaustion, and the single #21 UAT. Verify
changed surfaces first. Return to shape if the design requires dispatcher
repair, duplicate lifecycle logic, #21 changes, registries, or unrelated
full-suite ownership.
