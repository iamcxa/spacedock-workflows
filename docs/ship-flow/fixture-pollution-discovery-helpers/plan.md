# Fixture-tree exclusion for discovery helpers — Cycle 3 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: use `superpowers:executing-plans` task-by-task; every code task also uses both TDD skills named below.

**Goal:** Make adopter and density discovery fail loudly on traversal errors while preserving the existing Bash 3.2 fixture-pruning primitive, then consume AUTH-1 with one post-review repository-root run.

Each caller owns bounded temporary capture, validates producer status before consuming data, buffers accepted output until all traversals succeed, and normalizes operational failures to exit 2 with diagnostics and no stdout.
